# AGENTS.md

## 프로젝트 설명
- 이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다.
- 주요 목적은 CSV 파일의 거래 데이터를 읽고, 가계부 조회 및 요약 기능을 CLI로 제공하는 것이다.

## 코딩 규칙
- 모든 함수와 메서드에는 타입 힌트를 반드시 작성한다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.
- 구현은 읽기 쉽고 단순한 흐름을 우선한다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 기능 변경은 테스트 작성 -> 실패 확인 -> 구현 -> 리팩터 순서로 진행한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.
- 복잡도가 높아질 것 같으면 함수를 분리한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.
- 검수 결과에서 테스트 누락, 타입 힌트 누락, 복잡도 초과, 유지보수성 저하를 확인한다.

## 테스트 및 정적분석 명령어
- 테스트 실행: `pytest`
- 순환복잡도 점검: `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 즉시 커밋하고 푸시한다.
- 기능 단위로 커밋을 분리한다.
- 커밋 전에는 반드시 테스트와 품질 검수를 완료한다.
