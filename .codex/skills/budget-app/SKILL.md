# budget-app

## 언제 사용하나
- CSV 기반 가계부 CLI 앱을 구현, 수정, 검수할 때 사용한다.
- 이 저장소의 개발 규칙, 테스트 흐름, 품질 기준을 빠르게 확인할 때 사용한다.

## 핵심 규칙
- 구현보다 먼저 테스트를 작성한다.
- 모든 함수와 메서드에 타입 힌트를 넣는다.
- 함수는 최대 50줄로 유지한다.
- 순환복잡도는 10 이하로 유지한다.
- 커밋 전에는 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.

## 권장 흐름
1. 관련 테스트를 먼저 추가하거나 갱신한다.
2. 구현 코드를 작성한다.
3. `pytest`로 검증한다.
4. `radon cc`로 복잡도를 확인한다.
5. `qa_engineer` 관점으로 누락과 리스크를 다시 점검한다.

## 이 저장소의 성격
- 입력 데이터는 CSV 파일이다.
- CLI에서 동작하는 가계부 도구를 목표로 한다.
- 작은 함수와 명확한 책임 분리를 우선한다.
